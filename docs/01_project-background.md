# Project Background

## 1. 프로젝트 개요

본 프로젝트는 상업 화물 운송 및 통관 업무에서 반복적으로 작성되는 고객 안내 이메일을 표준화하기 위해 개발한 웹 기반 이메일 드래프트 생성기입니다.

운송 진행 상황, 통관 완료, 인보이스 안내, 본드 갱신, DOA 승인 안내 등 고객에게 자주 전달해야 하는 이메일을 담당자가 매번 직접 작성하지 않고, 정해진 입력값과 선택 항목을 바탕으로 일관된 형식의 이메일 초안으로 생성할 수 있도록 구성했습니다.

이 도구는 팀 담당자들이 고객에게 보다 전문적이고 통일된 안내를 제공할 수 있도록 하기 위해 개발한 내부 생산성 도구입니다.

---

## 2. 개발 배경

상업 화물 운송 및 통관 업무에서는 고객에게 반복적으로 안내해야 하는 이메일이 많습니다.

예를 들어 다음과 같은 안내가 자주 발생합니다.

- PRE-ALERT 안내
- 통관 완료 및 출고 정보 확인 안내
- 운송 인보이스 안내
- 본드 갱신 인보이스 안내
- DOA 승인 안내
- CARM, RPP, POA, CO, HS Code 관련 필수 안내

이러한 이메일은 기본 구조가 반복되지만, 실제 작성 시에는 화주명, BL 번호, ETA, 수입자명, 갱신일, 주소, 연락처 등 상황별 정보가 달라집니다.

또한 CARM, RPP, POA, Certificate of Origin, HS Code 등 통관 및 수입 진행과 관련된 필수 안내 항목은 누락될 경우 고객의 업무 진행에 영향을 줄 수 있습니다.

따라서 단순한 이메일 작성 보조를 넘어, 실무상 필요한 안내 항목을 빠짐없이 포함하고, 담당자별 문구 차이를 줄이며, 고객 커뮤니케이션의 품질을 일정하게 유지할 수 있는 도구가 필요했습니다.

또한 본 프로젝트는 향후 Progress Sheets 자동화와의 연계를 염두에 두고 기획되었습니다.

별도로 진행한 Progress Sheets 자동화는 운송 및 통관 진행 상황을 시트 기반으로 관리하고, 주요 일정과 상태 정보를 자동으로 업데이트하는 구조입니다. 이메일 드래프트 생성기는 이러한 진행 관리 자동화와 연결되어, 특정 업무 마일스톤에 도달했을 때 필요한 고객 안내 이메일을 보다 빠르게 생성할 수 있도록 하는 확장 가능성을 고려해 설계되었습니다.

예를 들어 화물 도착 예정, 창고 입고 예정, 통관 완료, 인보이스 안내, 픽업 또는 배송 확인, 본드 갱신 등 주요 단계에서 필요한 이메일을 Progress Sheets의 상태 정보와 연결하면, 담당자는 시트에 정리된 정보를 바탕으로 표준화된 Outlook 이메일 드래프트를 자동 또는 반자동으로 생성할 수 있습니다.

따라서 현재 버전은 단일 HTML 기반의 이메일 드래프트 생성 도구이지만, 장기적으로는 운송 진행 관리와 고객 커뮤니케이션을 하나의 자동화 워크플로우로 연결하는 기반 모듈을 목표로 설계했습니다.

## 3. 기존 업무상 문제점

기존 방식에서는 담당자가 이전 이메일을 복사하거나, 개별적으로 저장해 둔 문구를 조합해 이메일을 작성해야 했습니다.

이 방식에는 다음과 같은 한계가 있었습니다.

### 3.1 담당자별 문구 차이

같은 상황에 대한 안내라도 담당자마다 사용하는 문장, 표현, 형식이 달라질 수 있습니다.

고객 입장에서는 회사의 안내 톤이 일관되지 않게 느껴질 수 있으며, 내부적으로도 이메일 품질 관리가 어려워질 수 있습니다.

### 3.2 필수 안내 항목 누락 가능성

운송 및 통관 업무에서는 상황에 따라 CARM, RPP, POA, CO, HS Code 등 추가 안내가 필요할 수 있습니다.

하지만 수동 작성 방식에서는 필요한 항목을 빠뜨리거나, 오래된 문구를 그대로 사용할 가능성이 있습니다.

### 3.3 반복 작성으로 인한 시간 소요

PRE-ALERT, 통관 완료, 인보이스 안내 등은 자주 반복되는 업무입니다.

매번 기존 이메일을 찾아 복사하고, 필요한 부분을 수정하고, 문구를 다시 확인하는 과정은 작지만 누적되는 시간이 큽니다.

### 3.4 국문/영문 이메일 작성 부담

고객에 따라 국문 또는 영문 이메일이 필요합니다.

동일한 업무 상황에 대해 국문과 영문 문구를 별도로 관리해야 하므로, 담당자가 매번 번역하거나 이전 이메일을 찾는 방식은 비효율적입니다.

### 3.5 Outlook 붙여넣기 시 서식 유지 문제

업무 이메일은 대부분 Outlook에서 발송되므로, 앱에서 생성한 이메일을 복사해 Outlook에 붙여넣었을 때 볼드, 밑줄, 줄바꿈 등의 서식이 유지되는 것이 중요했습니다.

초기 구현에서는 일반 텍스트 복사 방식으로 인해 Outlook에 붙여넣을 때 서식이 사라지는 문제가 있었고, 이후 HTML 형식과 plain text를 함께 클립보드에 담는 방식으로 개선했습니다.

## 4. 프로젝트 목표

본 프로젝트의 목표는 다음과 같습니다.

### 4.1 이메일 작성 시간 단축

반복적으로 작성되는 고객 안내 이메일을 자동 생성함으로써 담당자의 수동 작성 시간을 줄이는 것을 목표로 했습니다.

담당자는 필요한 입력값을 기입하고 체크박스를 선택한 뒤, 생성된 이메일 본문을 복사해 Outlook에 붙여넣는 방식으로 빠르게 이메일을 작성할 수 있습니다.

### 4.2 고객 안내 품질 표준화

담당자별로 달라질 수 있는 문구와 형식을 표준화하여, 고객에게 일관된 톤과 구조의 안내 이메일을 제공하는 것을 목표로 했습니다.

이를 통해 고객은 회사로부터 보다 전문적이고 정돈된 안내를 받을 수 있습니다.

### 4.3 필수 안내 항목 누락 방지

CARM, RPP, POA, CO, HS Code 등 수입 및 통관 진행에 필요한 주요 안내 항목을 체크박스 방식으로 선택할 수 있도록 구성했습니다.

담당자는 상황에 따라 필요한 항목을 선택하고, 앱은 해당 항목에 맞는 안내 문구를 이메일 본문에 자동 삽입합니다.

### 4.4 국문/영문 이메일 관리 효율화

동일한 입력값을 바탕으로 국문 또는 영문 이메일을 생성할 수 있도록 구성하여, 이중 언어 이메일 작성 부담을 줄이고자 했습니다.

PRE-ALERT, 통관 완료 안내, 인보이스 안내 등 주요 템플릿은 국문/영문 전환이 가능하도록 설계되었습니다.

### 4.5 간단한 배포와 팀 공유

회사 팀원들과 함께 사용할 수 있는 도구라는 점을 고려하여, 별도의 설치가 필요한 데스크탑 앱보다는 웹앱 방식으로 개발했습니다.

특히 초기 단계에서는 단일 HTML 파일 구조를 사용하여, 별도의 서버나 복잡한 배포 절차 없이 브라우저에서 바로 실행할 수 있도록 했습니다.

### 4.6 Progress Sheets 자동화와의 연계 가능성 확보

본 프로젝트는 현재 독립적인 이메일 드래프트 생성기로 작동하지만, 향후 Progress Sheets 자동화와 연계하는 것을 염두에 두고 설계되었습니다.

Progress Sheets에는 화물별 운송 진행 상황, 통관 상태, ETA, 창고 입고, 인보이스, 픽업/배송 여부 등 업무상 중요한 상태 정보가 누적될 수 있습니다.

이러한 데이터와 이메일 템플릿 생성 로직을 연결하면, 특정 마일스톤에 도달했을 때 필요한 이메일 유형을 자동으로 식별하고, 해당 화물 정보가 반영된 Outlook 이메일 드래프트를 생성하는 방식으로 확장할 수 있습니다.

이를 통해 단순한 이메일 작성 시간 단축을 넘어, 운송 진행 관리와 고객 안내 업무를 하나의 연속된 자동화 흐름으로 연결하는 것이 장기적인 목표입니다.

## 5. 해결 방식

본 앱은 사용자가 오른쪽 입력 패널에서 필요한 정보를 입력하면, 왼쪽 미리보기 패널에 이메일 초안이 실시간으로 생성되는 구조입니다.

현재 버전에서는 담당자가 직접 입력값을 기입하고 체크박스를 선택하는 방식으로 이메일 초안을 생성합니다. 이는 실무자가 최종 내용을 검토하고 상황에 맞게 조정할 수 있도록 하기 위한 반자동 방식입니다.

향후에는 Progress Sheets 자동화에서 관리되는 화물별 진행 상태와 주요 일정 데이터를 입력값으로 활용하여, 이메일 작성에 필요한 기본 정보가 자동으로 채워지는 구조로 확장할 수 있습니다. 이 경우 담당자는 자동 생성된 Outlook 드래프트를 검토하고 필요한 부분만 수정한 뒤 발송하는 방식으로 업무를 진행할 수 있습니다.

기본적인 사용 흐름은 다음과 같습니다.

1. 이메일 유형 탭을 선택합니다.
2. 필요한 경우 국문 또는 영문 버전을 선택합니다.
3. 화주명, BL 번호, ETA, 수입자명, 갱신일 등 필요한 정보를 입력합니다.
4. 상황에 맞는 추가 안내 항목을 체크박스로 선택합니다.
5. 왼쪽 미리보기 영역에서 생성된 이메일을 확인합니다.
6. 복사 버튼을 눌러 Outlook에 붙여넣습니다.

이 구조를 통해 사용자는 이메일을 처음부터 작성하지 않고, 표준화된 초안을 빠르게 생성할 수 있습니다.

## 6. 주요 이메일 템플릿

현재 앱은 다음과 같은 이메일 유형을 지원합니다.

### 6.1 PRE-ALERT

수입 예정 화물의 운송 일정과 ETA 정보를 고객에게 안내하는 이메일입니다.

화주명, BL 번호, 밴쿠버 ETA, 창고 ETA 등을 입력하면 제목과 본문이 자동 생성됩니다.

또한 CARM, RPP, POA, CO, HS Code 등 수입 진행 전 확인이 필요한 항목을 선택적으로 본문에 추가할 수 있습니다.

### 6.2 통관 완료 안내

화물의 통관 완료 후 고객에게 최종 배송지 정보 확인, 보유 정보 컨펌, 픽업/배송 여부 확인 등을 요청하는 이메일입니다.

이메일 히스토리에 회신하는 상황을 고려하여 제목란 없이 본문만 생성되도록 구성했습니다.

### 6.3 운송 인보이스 안내

운송 인보이스 발행 후 고객에게 결제 및 후속 진행 사항을 안내하는 이메일입니다.

통관 안내, 배송, 픽업 여부에 따라 문구를 선택적으로 추가할 수 있습니다.

### 6.4 본드 갱신 인보이스 안내

1년 유효 기간의 본드 만료 및 갱신과 관련하여 고객에게 인보이스를 안내하는 이메일입니다.

수입자명과 만료/갱신일을 입력하면 관련 이메일 제목과 본문이 자동 생성됩니다.

### 6.5 DOA 승인 안내

통관 업무 위임을 위한 DOA 승인 절차를 고객에게 안내하는 이메일입니다.

국문과 영문 안내문을 함께 확인하고 복사할 수 있도록 구성했습니다.

## 7. 주요 기능

본 앱의 주요 기능은 다음과 같습니다.

- 입력값 기반 이메일 제목 및 본문 실시간 생성
- 이메일 유형별 탭 구조
- 국문/영문 이메일 템플릿 전환
- 체크박스 기반 추가 안내 문구 자동 삽입
- ETA 및 날짜 형식 자동 변환
- Outlook 붙여넣기용 HTML 서식 복사
- 이메일 본문 복사 버튼
- 각 탭별 입력값 초기화 기능
- 화면 폭이 좁을 때 사용할 수 있는 가로 드래그 탭 바

## 8. 기술적 접근

본 프로젝트는 별도의 서버나 데이터베이스 없이 브라우저에서 바로 실행할 수 있는 경량 웹 애플리케이션으로 구현했습니다.

초기 목적이 팀 내부에서 빠르게 공유하고 사용할 수 있는 생산성 도구였기 때문에, 설치 과정이나 서버 운영 없이 누구나 즉시 사용할 수 있는 구조를 우선적으로 고려했습니다.

이를 위해 HTML, CSS, Vanilla JavaScript를 기반으로 단일 HTML 파일 형태로 구현했으며, Outlook에서 이메일 서식을 유지한 채 사용할 수 있도록 Clipboard API를 활용했습니다.

기술 스택 선정 배경, 시스템 계층 구조, 데이터 흐름 및 컴포넌트 설계에 대한 자세한 내용은 **system-architecture.md**에서 설명합니다.

## 9. 기대 효과

본 프로젝트를 통해 기대할 수 있는 효과는 다음과 같습니다.

### 9.1 업무 시간 단축

담당자가 이메일을 처음부터 작성하거나 기존 이메일을 찾아 수정하는 시간을 줄일 수 있습니다.

반복되는 고객 안내 이메일일수록 시간 절감 효과가 커집니다.

### 9.2 고객 커뮤니케이션 품질 향상

표준화된 문구와 형식을 사용함으로써 고객에게 보다 전문적이고 일관된 안내를 제공할 수 있습니다.

### 9.3 업무 정확도 향상

필수 안내 항목을 체크박스 기반으로 선택하게 함으로써, 통관 및 수입 진행에 필요한 주요 정보가 누락될 가능성을 줄일 수 있습니다.

### 9.4 팀 단위 업무 표준화

개별 담당자의 작성 습관에 의존하지 않고, 팀 차원에서 동일한 기준의 고객 안내 이메일을 사용할 수 있습니다.

### 9.5 신규 담당자 온보딩 지원

업무에 익숙하지 않은 담당자도 표준화된 템플릿을 활용하여 주요 고객 안내 이메일을 작성할 수 있습니다.

이를 통해 신규 담당자의 업무 적응에도 도움이 될 수 있습니다.

## 11. 향후 개선 가능성

향후에는 본 이메일 드래프트 생성기를 별도로 진행한 Progress Sheets 자동화와 연계하여, 운송 진행 상황 및 주요 일정 마일스톤에 따라 Outlook 이메일 드래프트를 자동 또는 반자동으로 생성하는 방향으로 확장할 수 있습니다.

현재 Progress Sheets 자동화는 화물별 진행 상황, ETA, 창고 입고, 통관 완료, 인보이스, 픽업/배송 여부 등 주요 상태 정보를 관리하는 기반이 될 수 있습니다. 이메일 드래프트 생성기는 이러한 상태 정보를 바탕으로 고객에게 필요한 안내 이메일을 생성하는 커뮤니케이션 자동화 모듈로 활용될 수 있습니다.

예상 가능한 확장 흐름은 다음과 같습니다.

1. Progress Sheets에서 화물별 진행 상태와 주요 날짜 정보를 관리합니다.
2. 특정 마일스톤에 도달한 건을 자동으로 식별합니다.
3. 해당 마일스톤에 맞는 이메일 템플릿을 선택합니다.
4. 시트의 화물 정보, ETA, BL 번호, 수입자명, 주소, 연락처 등을 이메일 템플릿에 자동 반영합니다.
5. Outlook 이메일 드래프트를 자동 생성합니다.
6. 담당자가 최종 검토 후 고객에게 발송합니다.

추가로 다음과 같은 기능 개선도 고려할 수 있습니다.

- Progress Sheets 데이터와 이메일 템플릿 자동 매핑
- 주요 마일스톤별 이메일 발송 후보 자동 표시
- Outlook 드래프트 생성 자동화
- 템플릿 내용을 별도 JSON 파일로 분리
- HTML, CSS, JavaScript 파일 분리
- 사용자별 자주 쓰는 문구 저장 기능
- 이메일 제목까지 복사하는 옵션 추가
- 템플릿 검색 기능
- 고객 유형별 문구 선택 기능
- 다크 모드 지원
- GitHub Pages 또는 사내 웹페이지 배포
- React 또는 Flutter Web 기반 재구성

다만 현재 버전은 설치 없이 바로 사용할 수 있는 단일 HTML 기반의 경량 도구라는 점에 초점을 두고 있습니다. 이는 향후 자동화 연계 전 단계에서 이메일 문구와 템플릿 로직을 먼저 표준화하기 위한 기반 작업으로 볼 수 있습니다.

## 12. 요약

본 이메일 드래프트 생성기는 상업 화물 운송 및 통관 업무에서 반복적으로 발생하는 고객 안내 이메일 작성을 표준화하고, 담당자의 업무 시간을 줄이며, 고객 커뮤니케이션의 전문성과 일관성을 높이기 위해 개발한 웹 기반 내부 생산성 도구입니다.

단순한 이메일 템플릿 모음이 아니라, 실제 실무 흐름에 맞춰 입력값, 체크박스, 언어 전환, 서식 복사 기능을 결합한 업무 개선형 프로젝트입니다.
